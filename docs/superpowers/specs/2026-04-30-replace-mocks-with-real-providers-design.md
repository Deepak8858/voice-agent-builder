# Design: Replace Mocks with Real Providers

**Date:** 2026-04-30
**Status:** Approved
**Changes:** none yet

---

## Overview

Replace mock/stub adapters with real implementations. Three independent subsystems:

1. **Auth** → real Clerk (env var swap, no code)
2. **LLM** → new Azure AI Foundry adapter for Kimi 2.6 (OpenAI-compatible)
3. **Voice** → implement real Vapi adapter

---

## 1. Auth — Real Clerk

**Current:** `mock-auth.service.ts` (raw user ID in cookies, no signing)
**Target:** `clerk-auth.service.ts` (JWT verification, Clerk SDK)

**What changes:**
- Add `CLERK_SECRET_KEY` to `.env` (already has the env var defined)
- `auth.module.ts` already uses `AUTH_PROVIDER=clerk` → ClerkAuthService
- No code changes needed

**DI already wired:**
```
AuthModule → useFactory: AUTH_PROVIDER === 'clerk' ? clerk : mock
```

**Edge cases:**
- `CLERK_SECRET_KEY` missing → ClerkAuthService logs warning, rejects all requests
- Dev without Clerk key → set `AUTH_PROVIDER=mock` temporarily

**Testing:**
- Add integration test for Clerk token verification
- Test provisioning flow (new user → upsert in DB)

## 2. LLM — Azure AI Foundry + Kimi 2.6

**Current:** `MockAgentGeneratorService` (keyword heuristics)
**Target:** New `AzureAiFoundryAdapter` extending `OpenAiLlmAdapter` pattern

**Endpoint:**
```
Base URL: https://deepak7238kgs-0666-resource.services.azureai.net/openai/v1
Model:    kimi-2.6-flash (or whatever the deployment name is)
```

**What changes:**

### New file: `apps/api/src/llm/adapters/azure-aifoundry.adapter.ts`

```typescript
@Injectable()
export class AzureAiFoundryAdapter implements LlmAgentGenerator {
  readonly name = 'azure-aifoundry';
  private readonly logger = new Logger(AzureAiFoundryAdapter.name);
  private readonly endpoint: string;
  private readonly model: string;
  private readonly apiVersion = '2024-02-01'; // Azure AI Foundry API version

  constructor(private readonly mock: MockAgentGeneratorService) {
    this.endpoint = env.LLM_BASE_URL ?? 'https://deepak7238kgs-0666-resource.services.ai.azure.com/openai/v1';
    this.model = env.LLM_MODEL ?? 'kimi-2.6-flash';
  }

  async generate(input: GenerateAgentDto): Promise<GenerateAgentResult> {
    if (!env.LLM_API_KEY) {
      this.logger.warn('LLM_API_KEY not set — falling back to mock.');
      return this.mock.generate(input);
    }

    const baseTemplate = this.pickTemplate(input);
    try {
      const spec = await this.callModel(input, baseTemplate);
      const parsed = AgentSpecSchema.safeParse(spec);
      if (!parsed.success) {
        this.logger.warn(`Azure AI Foundry returned invalid spec: ${parsed.error.message}`);
        return this.mock.generate(input);
      }
      return {
        spec: parsed.data,
        suggested_name: parsed.data.name,
        rationale: `Generated by Azure AI Foundry Kimi 2.6 (${this.model}). Seeded with template "${baseTemplate.name}".`,
        matched_template_slug: baseTemplate.slug,
      };
    } catch (err) {
      this.logger.warn(`Azure AI Foundry call failed: ${(err as Error).message}. Falling back to mock.`);
      return this.mock.generate(input);
    }
  }
  // callModel, buildSystemPrompt, buildUserPrompt, pickTemplate same as OpenAI adapter
}
```

### env.ts changes

Add new env vars:
```typescript
LLM_API_KEY: z.string().optional(),        // Azure AI Foundry API key
LLM_PROVIDER: z.enum(['mock', 'github', 'openai', 'anthropic', 'azure-aifoundry']).default('mock'),
```

### LLM module / DI changes

Update LLM module to register the new adapter and switch via `LLM_PROVIDER`.

**Edge cases:**
- Network failure → fall back to MockAgentGeneratorService
- Malformed JSON response → fall back to mock
- Invalid AgentSpecSchema → fall back to mock
- API key missing → fall back to mock

**Testing:**
- Unit test: mock successful response, verify parsed output
- Unit test: mock 401 error, verify fallback to mock generator
- Unit test: mock invalid JSON, verify fallback
- Unit test: mock valid JSON but invalid schema, verify fallback

## 3. Voice — Real Vapi Adapter

**Current:** `vapi.adapter.ts` — stub that throws on every method
**Target:** Full implementation using Vapi SDK

**What changes:**

### New file: `apps/api/src/voice/adapters/vapi.adapter.ts`

Full implementation of `VoiceRuntimeProvider` interface:
- `createAgent()` — POST to Vapi to create outbound voice agent
- `updateAgent()` — PATCH to update existing agent
- `createBrowserTestSession()` — Create a test session for browser-based testing
- `startOutboundCall()` — Initiate outbound call
- `transferCall()` — Transfer active call
- `endCall()` — End call
- `getTranscript()` — Fetch transcript after call
- `getRecording()` — Fetch recording URL

**Vapi SDK:** Use `@vapi-ai/server-sdk` or HTTP calls to `https://api.vapi.ai`

**.env changes:**
```
VAPI_API_KEY=your_key_here
VOICE_PROVIDER=vapi
```

**Edge cases:**
- Vapi returns error → wrap in AppError with provider context
- Call not found → 404 AppError
- Network timeout → retry once, then throw
- Missing VAPI_API_KEY → AppError (existing behavior but better error message)

**Testing:**
- Mock Vapi HTTP responses for each method
- Test error handling for 4xx/5xx responses
- Test transcript parsing with various formats

## 4. Tests — Replace vi.fn() Mocks with Real DB

**Current:** 13 test files using `vi.fn()` to mock Prisma client
**Target:** Tests using real Prisma with a test database

**Strategy:** Use a separate test database (Supabase preview branch or local Docker Postgres)

**Changes:**
- Add `DATABASE_URL_TEST` env var
- In test setup: connect Prisma to test DB, run migrations
- Replace `vi.fn()` mock patterns with real service calls
- Keep tests fast by isolating per-file (not full integration)

**Files to update:**
- `analytics.test.ts` → real AnalyticsService with test DB
- `billing.service.test.ts` → real BillingService with test DB
- `knowledge.test.ts` → already close, verify
- `mock-generator.test.ts` → already real (no DB needed)

**Edge cases:**
- Test DB not available → skip tests with clear message
- Migration drift → run `prisma migrate deploy` in test setup
- Parallel test runs → use unique DB per test worker

## Rollback Plan

If any provider fails:
- Auth: flip `AUTH_PROVIDER=mock` in `.env`
- LLM: flip `LLM_PROVIDER=mock` in `.env`
- Voice: flip `VOICE_PROVIDER=mock` in `.env`
- Each provider has fallback to mock built into the adapter pattern
