# 25 — Risk Register

| Risk | Impact | Mitigation |
|---|---|---|
| Crowded market | High | Win with templates, agency white-label, compliance, UX |
| Outbound legal risk | Very high | Start inbound, require consent, DNC/DND, opt-out |
| Voice reliability | High | Test scenarios, fallback, human transfer, evaluations |
| Latency | High | Use proven providers first, track latency, keep prompts short |
| Provider lock-in | Medium | Adapter layer and neutral Agent Spec JSON |
| Tool mistakes | High | Schema validation, confirmation, logs, idempotency |
| Cross-tenant leak | Very high | Workspace guards and permission tests |
| High cost | Medium | Usage billing, cost tracking, provider comparison |
| Abuse/spam | Very high | Rate limits, compliance gate, campaign review |
| Building too much | High | MVP discipline and agency wedge |

## Biggest Product Risk
Building a beautiful demo that fails on real phone calls. Reliability, fallback, and analytics matter more than visual polish.
